# Сайт ТОО «СХМ Агро»

Производство, продажа и сервис сельхозтехники в Казахстане: каталог, новости,
услуги, приём заявок, панель управления и ИИ-помощник.

Без базы данных: всё хранится в одном файле `data/store.json`. Ставить СУБД не
нужно, а когда понадобится — переписывается один файл `server/store.js`,
эндпоинты остаются как есть.

---

## Запуск на своей машине

```bash
npm install
npm run init-db      # создаёт data/store.json с начальными данными
npm run dev:server   # API на 3001
npm run dev:client   # сайт на 5173
```

Нужен Node 22.9 или новее: сервер запускается с флагом `--env-file-if-exists`,
которого в более старых версиях нет.

Панель управления — `/admin`, пароль по умолчанию `admin` (меняется через
`ADMIN_PASSWORD`).

## Запуск на сервере

Docker, автоподъём после сбоя и ежесуточные бэкапы — в
[DEPLOY-DOCKER.md](DEPLOY-DOCKER.md).
Развёртывание на Railway — в [DEPLOY.md](DEPLOY.md).

### Где размещать: сервер должен стоять в Казахстане

Сайт принимает заявки с ФИО и телефонами — это персональные данные граждан
РК. Статья 12 закона № 94-V «О персональных данных и их защите» требует
хранить их в базе, физически расположенной на территории Казахстана; норма
действует с 2016 года. За нарушение предусмотрены штрафы и блокировка сайта.

Отсюда практический вывод: **Railway и DigitalOcean для боевого запуска не
подходят** — их серверы за границей. Показать сайт заказчику на них можно,
принимать настоящие заявки — нет. Для боевого запуска нужен казахстанский
хостинг (hoster.kz, ps.kz, Akashi и подобные): берётся облачный сервер с
Ubuntu и root-доступом, дальше всё по той же инструкции — Docker везде
одинаковый.

Отправка данных за границу — отдельный вопрос (трансграничная передача).
Поэтому в ИИ-сервисы из заявок не уходят ни ФИО, ни телефоны: туда
отправляются только тип обращения, техника, регион и комментарий.

Это не юридическая консультация — перед запуском подтвердите требования у
юриста.

## Что сделано по безопасности

Проверено запуском в боевом режиме, а не по описанию:

- вход в панель по паролю, сессия подписана и живёт 12 часов; смена пароля
  обрывает все активные сессии;
- пароль и подпись сверяются за постоянное время;
- ограничение частоты: 5 заявок за 10 минут с одного адреса, 10 попыток входа
  за 15 минут, 20 сообщений в чат за минуту. За обратным прокси считается
  настоящий адрес клиента, а не адрес прокси;
- заголовки CSP, HSTS, `X-Frame-Options: DENY`, `nosniff`, `Referrer-Policy`;
  версия сервера наружу не сообщается;
- согласие на обработку персональных данных проверяется на сервере, а не
  только галочкой в форме; вместе с заявкой сохраняется время согласия и
  редакция политики;
- ФИО и телефоны клиентов не отправляются в ИИ-сервисы;
- в образ Docker не попадают ни `.env`, ни ключи; процесс работает не от root.

Политика конфиденциальности — на странице `/privacy`.

## Перед запуском для клиентов

Это то, что нельзя сделать за вас: нужны ваши данные и решения.

### Обязательно

- [ ] **Сменить пароль администратора.** Пока он `admin`, в панель войдёт кто
      угодно и удалит каталог, новости и заявки. Сервер предупреждает об этом
      в логе при каждом запуске.
- [ ] **Задать `JWT_SECRET`** — иначе вход слетает при каждом перезапуске.
      Сгенерировать: `openssl rand -hex 32`
- [ ] **Включить HTTPS.** Пока сайт по `http://`, пароль администратора идёт
      по сети открытым текстом. С доменом включается одной командой —
      см. DEPLOY-DOCKER.md, раздел 6.
- [ ] **Проверить, что бэкапы делаются:** `ls backups/` через сутки после
      запуска. И хотя бы раз попробовать восстановление — бэкап, который
      никогда не восстанавливали, это надежда, а не резервная копия.
- [ ] **Забирать копии за пределы сервера.** Всё, что лежит на droplet,
      исчезнет вместе с droplet.

### Содержание сайта

- [ ] **Реквизиты в политике конфиденциальности.** В разделе 1 нужны БИН и
      полное юридическое наименование — в настройках сайта их нет, я не стал
      выдумывать. Текст политики — рабочая заготовка; перед публикацией его
      должен посмотреть юрист.
- [ ] **Проверить сроки хранения данных** (раздел 7 политики): написано «пока
      идёт сделка и действует гарантия». Если у вас иначе — поправьте, документ
      должен описывать то, что вы делаете на самом деле.
- [ ] **Контакты в настройках.** Телефон, почта и адрес сейчас взяты из
      исходного макета — проверьте, что они настоящие.
- [ ] **Ссылки на соцсети** в настройках. Пока поля пустые, значки в подвале
      не показываются — это нормально, но лучше заполнить.
- [ ] **Фотографии.** Снимки техники и цеха сейчас стоковые и не ваши.
      Раздел «Цех, а не перепродажа» держится на тексте; с настоящими фото
      сборки, покраски и полигона он станет заметно сильнее.
- [ ] **Тексты трёх этапов производства** написаны мной по вашей же фразе
      «льём узлы, собираем, красим и обкатываем». Прочитайте: это утверждения
      о вашем производстве, отвечать за них вам.
- [ ] **Сертификаты** — сейчас это карточки с названиями, без самих документов.

### Стоит знать

- Ключи ИИ в `.env` — тестовые. Расход на боевом сайте: примерно $2–5 в месяц
  при 100 сообщениях в чат в день. Ответы кэшируются, повторные вопросы и
  неизменившиеся заявки бесплатны.
- Загрузки своих фотографий через панель пока нет — список снимков задан в
  коде (`src/components/AdminForms.jsx`).
- Ключ Figma в `.mcp.json` стоит отозвать, если он ещё действует.
